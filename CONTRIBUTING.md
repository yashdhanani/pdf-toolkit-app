# Contributing

Thank you for your interest in contributing to PDF Toolkit!

## How to Contribute

1. **Fork the repository** on GitHub.
2. **Create a new branch** for your feature or bugfix:

```bash
git checkout -b feature/your-feature-name
```

3. **Make your changes** and commit them:

```bash
git commit -m "Add your descriptive commit message"
```

4. **Push to your fork**:

```bash
git push origin feature/your-feature-name
```

5. **Open a Pull Request** with a clear description of your changes.

## Reporting Bugs

Found a bug? Please open a GitHub Issue with:

- **Description**: What is the bug?
- **Steps to Reproduce**: How can we reproduce it?
- **Expected Behavior**: What should happen?
- **Actual Behavior**: What actually happens?
- **Screenshots**: If applicable, add screenshots or screen recordings.
- **Environment**: Browser, OS, etc.

## Suggesting Features

Have a feature idea? Open a GitHub Issue with:

- **Problem**: What problem does this solve?
- **Proposed Solution**: How would you solve it?
- **Alternative Solutions**: Any other approaches?
- **Additional Context**: Any other information that might be useful.

## Coding Style

- Follow the existing code formatting and style.
- Keep code readable and well-commented.
- Write meaningful commit messages.
- Test your changes before submitting a PR.
- Ensure no console errors or warnings are introduced.

## Development Setup

1. Clone your fork:

```bash
git clone https://github.com/YOUR_USERNAME/pdf-toolkit-app.git
cd pdf-toolkit-app
```

2. Install dependencies:

```bash
npm install
```

3. Start the development server:

```bash
npm run dev
```

4. Build for production:

```bash
npm run build
```

## Pull Request Guidelines

- Describe what your PR does and why.
- Link to any related issues.
- Keep PRs focused—one feature or bug fix per PR.
- Ensure the build passes before requesting review.
- Be open to feedback and iteration.

## Code of Conduct

Please be respectful and follow our Code of Conduct. We are committed to maintaining a welcoming and inclusive community.
